# 泰剧 AI 配音流水线

用 AI 技术替代/辅助传统人工配音，降低成本、提升效率。

## 背景

泰剧引进中国的配音流程：

```
泰剧原片（可能附带泰语字幕）→ 我们出中文配音台本 → 送广电审核 → 审核通过 → 严格按台本配音（一字不改）→ 成片
```

我们负责**台本制作 + 配音**两个环节。片方可能直接提供泰语字幕，也可能需要我们自行识别。

核心约束：**送审的台本就是最终版，审核通过后一个字都不能改，配音必须严格按台本执行。**

因此，送审前的台本必须一步到位，同时满足：
1. 语义准确（通过广电审核）
2. 口语化（适合配音演员/TTS朗读）
3. 字数控制（匹配原片每句时长，减少后期音频拉伸）
4. 经过试合成验证（送审前确认 TTS 效果可控）

## 配音台本的特殊要求

配音台本不是简单的字幕翻译，送审稿必须直接达到配音质量：

| 维度 | 普通字幕 | 配音台本（送审稿=最终稿） |
|------|----------|--------------------------|
| 语体 | 书面语、精简 | 口语化、自然（聆听性） |
| 字数约束 | 固定上限（每行≤N字） | 动态匹配原片该句时长 |
| 起止对齐 | 不需要 | 需要（台词起止与演员说话起止大致对齐） |
| 角色一致性 | 弱 | 严格（角色小传 + 语言风格） |
| 消费者 | 读者（可回看） | 配音演员 / TTS 引擎（一遍过） |

> **关于口型匹配**：泰剧是外语引进片，观众不期待完美口型同步。只需做到台词起止时间与演员说话时间大致对齐即可，不需要做到双唇音帧级匹配。精确口型匹配的性价比很低，不作为硬性要求。

### 口语化要求（聆听性）

- 使用语气词：啊、嗯、哎、吧、呢、啦
- 口语化表达、缩略语、自然停顿
- 观众只听一遍没机会回看，必须一听就懂

### 角色小传

每个角色需维护：性格概述、说话习惯、口头禅、语域层次，全剧保持一致。

### 非对白音频处理

泰剧中除了对白，还有其他类型的音频需要区分处理：

| 类型 | 处理方式 |
|------|----------|
| 对白 | AI 配音（核心流程） |
| 旁白/独白 | AI 配音，使用单独的旁白音色 |
| 画外音/电话声 | AI 配音，需加音效后处理（如电话滤波） |
| 重叠对白（多人同时说话） | 分别合成后叠加混音，需注意时间交错和音量平衡 |
| 歌曲/片头片尾曲 | 通常保留原声，不配音 |
| 背景人声/群杂 | 保留原声或用中文群杂替换 |

### 送审版 vs 配音版

台本需要输出两种格式：

| | 送审版 | 配音版 |
|---|---|---|
| 内容 | 时间码 + 角色名 + 台词 | 时间码 + 角色名 + 台词 + 情感标注 |
| 用途 | 广电审核（只看内容） | 指导配音（含情绪、语速、气口提示） |
| 台词内容 | 完全一致 | 完全一致 |

情感标注（如"冷笑""焦急催促"）是给配音用的技术指令，不应送审，避免造成困惑。两个版本的台词文本完全一致，只是配音版多了标注信息。

### 成片交付格式

| 交付物 | 格式 | 说明 |
|--------|------|------|
| 完整成片 | MP4/MKV（替换音轨） | 最终播出版本 |
| 中文配音轨 | WAV/AAC | 单独的配音音频 |
| 背景音乐/音效轨 | WAV/AAC | 从原片分离的背景音 |
| 配音台本 | 送审版 + 配音版 | 文档存档 |

> 具体交付格式需与播出平台确认。多轨交付便于后期调整，但部分平台只需要完整成片。

## TTS 方案调研

剧集配音对 TTS 的核心要求：语音克隆（用原演员音色说中文）、情感控制（哭/笑/怒/耳语）、中文自然度、时长可控、多角色一致性。

### Tier 1：最适合剧集配音

| 方案 | 语音克隆 | 情感控制 | 中文质量 | 价格 |
|------|----------|----------|----------|------|
| **Fish Audio S1/S2** | 10s 参考音频零样本，TTS-Arena #1 | 64+ 情感标签，逐句指定如 `(sad)(whispering)` | 极佳，30万+小时中文训练 | ~¥5/小时 |
| **CosyVoice 3 (阿里)** | 零样本跨语言克隆 | 指令式情感控制 | 原生优势，18种中文方言 | 开源免费 / 阿里云API |
| **Qwen3-TTS (通义)** | 3s 参考音频零样本 | 指令式情感合成 | 原生中文，Apache 2.0 | 开源免费 |

### Tier 2：特定场景有优势

| 方案 | 特点 | 适用场景 | 短板 |
|------|------|----------|------|
| **GPT-SoVITS v2ProPlus** | 微调后音色相似度最高 | 主角跨集一致性 | 需每角色单独训练，无时长控制 |
| **MegaTTS 3 (字节)** | 3s 零样本，0.45B 参数 | 快速原型验证 | 无情感控制 API |
| **ElevenLabs** | 端到端 Dubbing Studio | 商业快速交付 | 中文声调不如 Fish Audio，价格高 |

### Tier 3：不推荐用于剧集

| 方案 | 原因 |
|------|------|
| ChatTTS | 仅支持 `[laugh]` `[uv_break]` 等极少标签，情感不够 |
| F5-TTS | 无情感标签支持 |

### TTS 选型建议

> **POC 阶段建议先用一个方案跑通全流程**，避免多套系统音质风格不一致导致同一场对话听感割裂。推荐先用 **Fish Audio S1** 做全量验证，效果确认后再根据需要引入其他方案。

### 语音克隆的跨语言风险

用泰语原演员音色克隆后说中文（跨语言克隆），目前效果不稳定：
- 音色相似度会明显下降
- 中文发音可能不够自然
- 部分音色特征在跨语言时丢失

**备选方案**：从中文音色库中选取与原演员音色气质接近的声音，效果更稳定但失去原演员特征。POC 阶段应对两种方案做 A/B 对比。

### 时长控制策略

精确时长匹配是行业共同痛点，采用**翻译阶段前端控制 + 试合成验证 + 合成阶段后端微调**的策略：

1. **翻译阶段**：通过字数控制，让中文台词时长与原句偏差控制在 ±10% 以内
2. **试合成验证**：送审前用 TTS 试合成，发现偏差大的句子立即调整台本
3. **合成阶段**：TTS 语速参数粗调 + rubberband 微调（因已经过验证，此处偏差应很小）
4. **质量红线**：rubberband 拉伸幅度不超过 ±15%

```
翻译时字数控制 → 试合成验证（不达标→调台本）→ 送审 → 正式合成 → rubberband 微调
       ±10%            ↻ 迭代                              ≤±15%
```

值得关注：IndexTTS-2 声称支持毫秒级时长控制，但该功能尚未开源。

## 关键挑战

- **台本一步到位** — 送审稿必须同时满足审核要求和配音质量，没有二次修改机会
- **试合成与台本的迭代效率** — 试合成发现问题需快速改台本并重新验证，不能拖慢送审节奏
- **审核不通过的回路** — 修改台本时需同步更新送审版和配音版，并重新做试合成验证
- **跨语言语音克隆** — 泰语音色→说中文效果不稳定，需 POC 验证是否可行
- **多角色音色区分** — 每个角色需独立音色，全剧保持一致
- **情感表达** — 泰剧情感丰富（哭、笑、怒），TTS 需匹配情绪
- **重叠对白** — 多人同时说话场景的说话人分离和音频叠加
- **非对白处理** — 旁白、画外音、歌曲、群杂等需分类处理
- **背景音保留** — 干净分离人声，配音后混回原始背景音乐和音效

## 成本对比

| 项目 | 传统配音 | AI 配音 |
|------|----------|---------|
| 单集 (45min) | ¥5,000-15,000 | ¥2,000-5,000 |
| 制作周期 | 3-5 天/集 | 1-2 天/集 |

> AI 配音成本包含：台本人工精校、试合成迭代、角色音色准备与调试（10+角色）、TTS API/GPU 费用、混音质检人工。不含审核不通过的返工成本。
