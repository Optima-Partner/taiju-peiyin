# taiju-peiyin

泰剧 AI 配音流水线 — 用 AI 技术替代/辅助传统人工配音，降低成本、提升效率。

## 背景

泰剧引进中国的配音流程：

```
泰剧原片（可能附带泰语字幕）→ 我们出中文配音台本 → 送广电审核 → 审核通过 → 严格按台本配音（一字不改）→ 成片
```

我们负责**台本制作 + 配音**两个环节。片方可能直接提供泰语字幕，也可能需要我们自行识别。

核心约束：**送审的台本就是最终版，审核通过后一个字都不能改，配音必须严格按台本执行。**

因此，送审前的台本必须一步到位，同时满足：
1. 语义准确（通过广电审核）
2. 口语化（适合配音演员/TTS朗读）
3. 时长适配（匹配原片每句时长）
4. 情感标注（指导配音表演）

## 配音台本的特殊要求

配音台本不是简单的字幕翻译，送审稿必须直接达到配音质量：

| 维度 | 普通字幕 | 配音台本（送审稿=最终稿） |
|------|----------|--------------------------|
| 语体 | 书面语、精简 | 口语化、自然（聆听性） |
| 字数约束 | 固定上限（每行≤N字） | 动态匹配原片该句时长 |
| 口型同步 | 不需要 | 需要（起止对齐 + 关键唇形匹配） |
| 情感标注 | 无 | 逐句标注（情绪、语速、气口） |
| 角色一致性 | 弱 | 严格（角色小传 + 语言风格） |
| 消费者 | 读者（可回看） | 配音演员 / TTS 引擎（一遍过） |

### 口型匹配规则

- 中文台词的起止时间必须对齐演员嘴巴的开合
- 在演员做双唇音（p/b/m）处，中文尽量用双唇音（b/p/m/f）
- 在关键嘴型变化点匹配音节数，而非简单匹配总字数

### 口语化要求（聆听性）

- 使用语气词：啊、嗯、哎、吧、呢、啦
- 口语化表达、缩略语、自然停顿
- 观众只听一遍没机会回看，必须一听就懂

### 情感标注格式

每句台词附带：
- 核心情绪（温柔叮嘱、焦急催促、冷笑…）
- 气口/节奏提示（语速加快、停顿、叹气）
- 角色标识（颜色/编号区分）

### 角色小传

每个角色需维护：性格概述、说话习惯、口头禅、语域层次，全剧保持一致。

## TTS 方案调研

剧集配音对 TTS 的核心要求：语音克隆（用原演员音色说中文）、情感控制（哭/笑/怒/耳语）、中文自然度、时长可控、多角色一致性。

### Tier 1：最适合剧集配音

| 方案 | 语音克隆 | 情感控制 | 中文质量 | 价格 |
|------|----------|----------|----------|------|
| **Fish Audio S1/S2** | 10s 参考音频零样本，TTS-Arena #1 | 64+ 情感标签，逐句指定如 `(sad)(whispering)` | 极佳，30万+小时中文训练 | ~¥5/小时 |
| **CosyVoice 3 (阿里)** | 零样本跨语言克隆 | 指令式情感控制 | 原生优势，18种中文方言 | 开源免费 / 阿里云API |
| **Qwen3-TTS (通义)** | 3s 参考音频零样本 | 指令式情感合成 | 原生中文，Apache 2.0 | 开源免费 |

### Tier 2：特定场景有优势

| 方案 | 特点 | 适用场景 | 短板 |
|------|------|----------|------|
| **GPT-SoVITS v2ProPlus** | 微调后音色相似度最高 | 主角跨集一致性 | 需每角色单独训练，无时长控制 |
| **MegaTTS 3 (字节)** | 3s 零样本，0.45B 参数 | 快速原型验证 | 无情感控制 API |
| **ElevenLabs** | 端到端 Dubbing Studio | 商业快速交付 | 中文声调不如 Fish Audio，价格高 |

### Tier 3：不推荐用于剧集

| 方案 | 原因 |
|------|------|
| ChatTTS | 仅支持 `[laugh]` `[uv_break]` 等极少标签，情感不够 |
| F5-TTS | 无情感标签支持 |

### 推荐的混合策略

按角色重要性分配不同方案，平衡质量和成本：

```
角色分类              TTS 方案                    说明
─────────────────────────────────────────────────────────────
主角（3-5人）    →   GPT-SoVITS 微调             1min+ 参考音频训练
                                                 跨集音色一致性最高

配角（5-10人）   →   Fish Audio S1/S2 API        10s 参考音频零样本克隆
                                                 64+ 情感标签精细控制

龙套/路人        →   CosyVoice 3 / Qwen3-TTS    自部署，零成本，效果够用
```

### 时长控制：行业共同痛点

精确时长匹配（如"这句话用 3.42 秒说完"）目前所有方案都无法直接做到，实际做法：

```
TTS 生成（语速参数粗调）→ rubberband 音频时间拉伸（精调）→ 匹配原片时长
```

值得关注：IndexTTS-2 声称支持毫秒级时长控制，但该功能尚未开源。

## 关键挑战

- **台本一步到位** — 送审稿必须同时满足审核要求和配音质量，没有二次修改机会
- **多角色音色区分** — 每个角色需独立音色，全剧保持一致
- **情感表达** — 泰剧情感丰富（哭、笑、怒），TTS 需匹配情绪
- **时长匹配** — 中泰语速差异大，TTS 粗调 + rubberband 精调
- **口型同步** — 关键双唇音帧位的匹配
- **背景音保留** — 干净分离人声，配音后混回原始背景音乐和音效
