# taiju-peiyin

泰剧 AI 配音流水线 — 用 AI 技术替代/辅助传统人工配音，降低成本、提升效率。

## 背景

泰剧引进中国的配音流程：

```
泰剧原片（可能附带泰语字幕）→ 我们出中文字幕版 → 送广电审核 → 审核通过 → 台本改编 → 配音（不可改内容语义）→ 成片
```

我们负责**字幕制作 + 配音**两个环节。片方可能直接提供泰语字幕，也可能需要我们自行识别。

核心约束：
- **审核通过的字幕是语义锚点**，配音不可修改内容语义
- **配音台本是字幕的口语化改编**，措辞可调整但意思必须一致

## 配音台本 vs 普通字幕

配音台本不是简单的字幕翻译，有以下特殊要求：

| 维度 | 普通字幕 | 配音台本 |
|------|----------|----------|
| 语体 | 书面语、精简 | 口语化、自然（聆听性） |
| 字数约束 | 固定上限（每行≤N字） | 动态匹配原片该句时长 |
| 口型同步 | 不需要 | 需要（起止对齐 + 关键唇形匹配） |
| 情感标注 | 无 | 逐句标注（情绪、语速、气口） |
| 角色一致性 | 弱 | 严格（角色小传 + 语言风格） |
| 消费者 | 读者（可回看） | 配音演员 / TTS 引擎（一遍过） |

### 口型匹配规则

- 中文台词的起止时间必须对齐演员嘴巴的开合
- 在演员做双唇音（p/b/m）处，中文尽量用双唇音（b/p/m/f）
- 在关键嘴型变化点匹配音节数，而非简单匹配总字数

### 口语化要求（聆听性）

- 使用语气词：啊、嗯、哎、吧、呢、啦
- 口语化表达、缩略语、自然停顿
- 观众只听一遍没机会回看，必须一听就懂

### 情感标注格式

每句台词附带：
- 核心情绪（温柔叮嘱、焦急催促、冷笑…）
- 气口/节奏提示（语速加快、停顿、叹气）
- 角色标识（颜色/编号区分）

### 角色小传

每个角色需维护：性格概述、说话习惯、口头禅、语域层次，全剧保持一致。

## 核心技术栈

| 环节 | 技术方案 | 备选 |
|------|----------|------|
| 语音识别 (ASR) | Whisper large-v3 | Google Cloud STT |
| 字幕翻译 | Claude / GPT-4o | DeepL |
| 台本改编 | Claude / GPT-4o（口语化 prompt） | — |

| 语音合成 (TTS) | CosyVoice (阿里) | ChatTTS, MegaTTS |
| 口型同步（可选） | Wav2Lip / VideoReTalking | — |
| 音频处理 | FFmpeg, pydub | — |

## 关键挑战

- **两轮翻译的语义一致性** — 台本改编后意思不能偏离审核稿，需自动化校验
- **多角色音色区分** — 每个角色需独立音色，全剧保持一致
- **情感表达** — 泰剧情感丰富（哭、笑、怒），TTS 需匹配情绪
- **时间轴匹配** — 中泰语速差异大，需智能压缩/拉伸语音
- **口型同步** — 关键双唇音帧位的匹配
- **背景音保留** — 干净分离人声，配音后混回原始背景音乐和音效
